Phase A
🔧 Pre-Flight Checks
1
Confirm your folder has all 3 files
Open File Explorer → go to your OrderAgent folder (or wherever you saved the files). You must have:
OrderAgent/
  ├── product_order_agent_ollama.py      ✅ legitimate agent
  ├── vulnerable_agent_demo_ollama.py    ✅ injection demo
  └── credentials.json                  ✅ Gmail API credentials
If token.pickle is also there from a previous run, that's fine — it means Gmail is already authorized.
2
Fill in your credentials in the demo script required
Right-click vulnerable_agent_demo_ollama.py → Open with Notepad. At the top of the file, find these 2 lines and fill them in:
SENDER_EMAIL    = "yourname@gmail.com"      ← your Gmail address
SENDER_APP_PASS = "xxxx xxxx xxxx xxxx"    ← 16-char App Password
To get an App Password: go to myaccount.google.com → Security → 2-Step Verification → App passwords → Create one named "Order Agent"
Press Ctrl+S to save.
3
Confirm Ollama is running
Open Command Prompt and run:
ollama list
You should see llama3.1 listed. If you see nothing or an error, run:
ollama pull llama3.1     (one-time download ~4.7GB)
4
Open Command Prompt inside your folder
Open File Explorer → navigate to your OrderAgent folder → click the address bar at the top → type cmd → press Enter. A black Command Prompt window opens already pointing at your folder.
All commands from here on are run in this Command Prompt window.

Phase E
🛠️ Troubleshooting
!
❌ "No unread Product Order emails found"
The email must be unread in Gmail when the script runs.
Fix: In Gmail, right-click the email → "Mark as unread"
Then run the script again.
!
❌ "Cannot reach Ollama" or model not found
Fix 1: Open a new Command Prompt and run:  ollama serve
Fix 2: If model missing, run:  ollama pull llama3.1
!
❌ SMTP / email sending fails
Fix 1: Make sure you used an App Password, not your normal Gmail password
Fix 2: Go to myaccount.google.com → Security → 2-Step Verification
        must be ON before App Passwords work
Fix 3: Re-create the App Password and paste the new one into the script
!
❌ LLM calls 0 tools / ignores the injection
Fix 1: Use Attack Type 1 (most explicit — highest success rate)
Fix 2: Make sure you copied the injection text completely
Fix 3: Check the email body printed in the console matches what you sent
!
❌ Excel not found / not updating
Fix 1: Close the Excel file before running the script
Fix 2: Check EXCEL_PATH in the script matches where your file actually is
Fix 3: If the file doesn't exist yet, the script will create it automatically
!
❌ No files found by search_files tool
Fix: Open the script in Notepad, find:
  SEARCH_ROOT = r'C:\Users\Ashish\Desktop\IIJ AI\...'
Change it to a folder that actually contains your Excel file, e.g:
  SEARCH_ROOT = r'C:\Users\Ashish\Desktop'
